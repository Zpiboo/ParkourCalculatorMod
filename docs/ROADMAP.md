# Roadmap

Live status of upcoming work. Version numbers live in `CHANGELOG.md` (generated by release-please); this file only tracks what's done and what's next.

## Done

- [x] Multi-module restructure: Minecraft-free UI / data / selection code in `core/`; the original Fabric mod is now `loader-fabric-1.21.10/`.
- [x] Configurable keybind to open UI (default `K`, rebindable in Controls).
- [x] `loader-forge-1.8.9/` and `loader-forge-1.12.2/`: shared `core/` + a `forge-core/` module that hosts the LWJGL 2 / ImGui bootstrap and the 1.8.9-shaped sprint state machine.
- [x] CI: per-loader builds on PRs, three-jar release on `v*` tags via release-please.
- [x] Draggable start box (in-world drag to reposition the simulation origin).
- [x] Don't open the overlay while another MC screen (chat, pause, inventory) owns input.

## Next (Phase 1: Manual Input Planning)

- [ ] Color-coded boxes by movement state (grounded / airborne / landing). `BoxStyle` already centralises the constants; this is mostly a `BoxRenderer` / state-tag plumbing change.
- [ ] Save/load input sequences in mothball-string notation (Stratfinder / Mothball interop).
- [ ] Playback mode: execute a saved macro in-game and confirm it matches the simulation.
- [ ] Hitboxes instead of point-dots.
- [ ] Subtick visualization (Y - X - Z substep order).

## Later (Phase 2: Input Optimization)

See `docs/VISION.md` § Phase 2 for the algorithm shape (A* boundary corridor + multithreaded input bruteforce).

- [ ] Position bruteforcing: target a block, search inputs that land on it.
- [ ] Distance checks between simulated player hitbox and arbitrary block hitboxes (the bruteforcer's fitness function).
- [ ] UI for bruteforce parameters (trial count, tick depth, key probabilities, thread count, sync interval).

## Later (Phase 3: Autonomous Pathfinding)

See `docs/VISION.md` § Phase 3. Multi-jump route generation against the loaded world.

- [ ] `loader-neoforge-1.21/` (or similar modern loader). New port pressure will probably surface gaps in `core/.../ports/`.
